1. CODE FORMATTING AND ALIGNMENT

Use 4 spaces for indentation. Never use tabs.

Place opening and closing curly braces on new lines.

Align curly braces vertically.

Maintain consistent alignment across the file.

Use CTRL+K, CTRL+D before committing code.

Do not insert unnecessary blank lines between code statements.

---------------------------------------------------------------------------------------------

2. COMMENTS

Comments must include a clear description.

Comments must:

Start with a capital letter.

End with a period.

Use /* */ for multi-line comments.

Use // for single-line comments.

Have exactly one space after //.

Be placed on a separate line (never inline).

Add single-line comments only when necessary to explain intent.

Do not leave unused or commented-out code.

---------------------------------------------------------------------------------------------

3. NAMING CONVENTIONS

Use meaningful and descriptive names for:

Variables

Methods

Classes

Constants

Test cases

Class names must use PascalCase.

Method names must use PascalCase.

Method arguments must use camelCase.

Local variables must use camelCase (lowercase allowed).

Do not use underscores in identifiers.

Do not use Hungarian notation.

Avoid abbreviations.

Use searchable constant names.

Do not repeat logic unnecessarily.

Do not pass indexed values directly into methods.

Store indexed values in descriptive variables first.

Prefer string interpolation over string concatenation.

Use && instead of &.

Use || instead of |.

Avoid too many parameters in methods.

----------------------------------------------------------------------------------------------

4. CLEAN CODE ENFORCEMENT

Remove unused imports.

Delete unused code.

Do not use console.log.

Prefer ternary operators over if-else where applicable.

Prefer foreach over for loops when suitable.

-------------------------------------------------------------------------------------------------

5. VERSION CONTROL RULES

Branch Naming Format:
TesterName_ModuleName_TestCaseName_Version

Pull Request Rules:

PR title must match test case name and version.

Include Jira link in commit message.

Include test case description in commit message.

Pull latest changes before raising PR.

Attach screenshots of passing tests for main and version builds.
